This defines the custom keys being saved at each level for the custom powerup.

(shared)

cards
    - priority: (blank == none), high, medium, low, backburner
        expected sorting: high, medium low, none, backburner

board
    - project: id of project

organization
    - projects: list of projects with keys: id, name, code (billing code)
    - memberGroups: list of members with keys: id, fullName, username, group ((blank == none), pm, designer, artist, programmer, writer)


(private)

organization
    - user_token: saved user auth token
